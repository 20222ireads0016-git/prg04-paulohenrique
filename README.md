Ideia de Projeto 1:

Codnome: Vortex: EventGrid
O Epicentro da Organização para Eventos Dinâmicos

O Objetivo:
O Vortex é um orquestrador de eventos de alta performance. Sua missão é organizar o "caos" inerente a grandes convenções e encontros técnicos, transformando-os em um fluxo de dados estruturado. Através de ferramentas de credenciamento via QR Code, gestão de cronogramas em tempo real e mapas de expositores, o sistema garante que a energia do evento flua sem interrupções.


Ideia de Projeto 2:

Codnome: DraftNote
Do Esboço à Entrega: Gestão Inteligente para Processos Criativos

O Objetivo
O DraftNote é uma plataforma de gestão voltada para artistas e profissionais critivos que lidam com comissions (encomendas de artes). O objetivo é centralizar o fluxo de trabalho, desde o rascunho inicial até a entrega final, permitindo que o profissional gerencie prazos, revisões e pagamentos de forma transparente em todas as etapas do processo (lineart, pintura, render), permitindo ao cliente acompanhar o nascimento da obra em tempo "real".


Ideia de Projeto 3:

Codnome: Syncra
Sincronia Inteligente entre Trabalho, Estudo e Vida

O Objetivo:
O Sincra é um maestro da rotina do usuário, que com o auxlio de Inteligencia Artifical, atua como organizador do seu dia a dia, tomando base nas preferencias do usuário ele busca o busca o equilíbrio entre o rigor dos estudos, as demandas do trabalho e a importância da vida social. Através de algoritmos de aprendizado o Syncra ajusta prioridades e sugere horários otimizados para que o usuário alcance produtividade sem abrir mão do bem-estar.



Estrutura do Projeto:

O projeto está dividido em 2 pastas principais (Frontend, Backend).

Frontend: A pasta frontend possui toda a estrutura HTML, CSS, JavaScript do projeto.

Subpastas de Frontend:

infraestructure - Contém a base do site e arquivos globais;

assets - Estilos (CSS), fontes, imagens e scripts (JS);

pages - Páginas base, como a página inicial (index.html);

usuario - Módulo dedicado às funcionalidades do usuário (Organizando em subpastas);

components - Destinado a elementos de interface reutilizáveis (botões, cabeçalhos, rodapés) [Lembrar de questionar ao professor se essa pasta não deveria estar em Infraestructure...].

Backend: A pasta backend possui toda a estrutura do JAVA, já configurado o Spring bot, mavem, e temporariamente conectado a um banco de dados Postgre temporário.
